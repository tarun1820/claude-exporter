AI Conversation Bridge

Optional: bring your own API key for Anthropic, OpenAI, or Google Gemini — or point at a model running entirely on your own machine via Ollama — to enable AI-enhanced context extraction when bridging a conversation to another LLM. Pick which provider to use below; only the fields for the selected provider are used, only that provider's own API is ever contacted, and only when you explicitly enable "AI-enhanced extraction" on the Bridge page (never automatically). All keys are excluded from backup and diagnostics exports.

Runs entirely on your machine — nothing leaves it. Requires Ollama to be started with CORS allowed for this extension (see docs/INSTALL.md).

Backup & Restore

Save the extension's data — model snapshots, export history, and preferences — to a file you can restore later. Useful before uninstalling, or to move your data to another browser, device, or extension build.

Model Display

Which model name to show in the browse view's Model column. Bounced chats (where the model has changed) get a small "*" marker either way — the tooltip shows the other model.
